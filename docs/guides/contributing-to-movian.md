# Contributing to Movian

Guide for contributing to the Movian project.

## Ways to Contribute

- Report bugs
- Suggest features
- Write documentation
- Submit code patches
- Create plugins
- Test releases
- Help other users

## Getting Started

### Fork Repository

```bash
git clone https://github.com/andoma/movian.git
cd movian
```

### Build from Source

See [Build System](../installation/build-system.md) for platform-specific instructions.

### Create Branch

```bash
git checkout -b feature/my-feature
```

## Code Contributions

### Coding Standards

- Follow existing code style
- Add comments for complex logic
- Write clear commit messages
- Test your changes

### Commit Messages

```
Short summary (50 chars or less)

Detailed explanation if needed. Wrap at 72 characters.

- Bullet points for multiple changes
- Reference issues: Fixes #123
```

### Pull Requests

1. Push branch to your fork
2. Create pull request on GitHub
3. Describe changes clearly
4. Wait for review
5. Address feedback

## Documentation

### Writing Docs

- Use clear, concise language
- Include code examples
- Test examples work
- Follow existing format

### Documentation Structure

See [Documentation Standards](../meta/documentation-standards.md).

## Testing

- Test on multiple platforms
- Test edge cases
- Verify no regressions
- Update tests if needed

## Community

- Be respectful
- Help others
- Share knowledge
- Follow code of conduct
